# 1. 변수와 스코프⭐️

<details>
  <summary>(1) 스코프란?</summary>

  * **스코프(Scope)** : 변수에 접근할 수 있는 범위
</details>

<details>
  <summary>(2) 전역 스코프(Global Scope)</summary>

  * **어디서든 접근 가능한 변수**
  ```JavaScript
  // 전역 변수
  let globalVar = '전역 변수';

  function myFunction() {
      console.log(globalVar); // 접근 가능
  }

  if (true) {
      console.log(globalVar);  // 접근 가능
  }

  console.log(globalVar); // 접근 가능
  ```

  * **장점과 단점**
  ```JavaScript
  // 장점: 어디서든 사용 가능
  let userName = '홍길동';

  function greet() {
      console.log('안녕하세요, ' + userName);
  }

  function showProfile() {
      console.log('프로필: ' + userName);
  }

  // 단점: 변경되면 모든 곳에 영향
  userName = '김철수';
  greet();          // 안녕하세요, 김철수
  showProfile();    // 프로필: 김철수

  // 단점: 이름 충돌 위험
  let count = 10;   // 전역
  // ... 1000줄 후
  let count = 20;   // 에러: 이미 선언됨
  ```
    
</details>

<details>
  <summary>(3) 함수 스코프(Function Scope)</summary>

  * **var는 함수 스코프를 따름**
  ```JavaScript
  function myFunction() {
      var funcVar = '함수 변수';
      console.log(funcVar);  // 접근 가능
  }

  myFunction();
  console.log(funcVar);  // ReferenceError
  ```

  * **if, for 등의 블록은 무시**
  ```JavaScript
  function test() {
      if (true) {
          var message = '안녕';
      }
      console.log(message); // 안녕(블록 밖에서도 접근 가능)
  }

  test();

  // for문도 마찬가지
  function loop() {
      for (var i = 0; i < 3; i++) {
          // 반복
      }
      console.log(i);  // 3(for문 밖에서도 접근)
  }

  loop();
  ```

  * **문제가 되는 경우**
  ```JavaScript
  var result = [];

  for (var i = 0; i < 3; i++) {
      result.push(function() {
        console.log(i);
      });
  }

  result[0]();  // 3 (예상: 0)
  result[1]();  // 3 (예상: 1)
  result[2]();  // 3 (예상: 2)

  // 모두 3이 나오는 이유:
  // i가 함수 스코프라서 하나만 존재
  // 반복문이 끝나면 i = 3
  ```

</details>

<details>
  <summary>(4) 블록 스코프(Block Scope)</summary>

  * **let과 const는 블록 스코프를 따름**
  ```JavaScript
  if (true) {
      let blockVar = '블록 변수';
      console.log(blockVar);  // 접근 가능
  }

  console.log(blockVar);  // ReferenceError
  ```

  * **올바른 예제**
  ```JavaScript
  let result = [];

  for (let i = 0; i < 3; i++) {
      result.push(function() {
          console.log(i);
      });
  }

  result[0]();  // 0
  result[1]();  // 1
  result[2]();  // 2

  // 각 반복마다 새로운 i가 생성됨
  ```

</details>

<details>
  <summary>(5) 스코프 체인(Scope Chain)</summary>

  * **안쪽에서 바깥쪽으로 찾아감**
  ```JavaSCript
  let global = '전역';

  function outer() {
      let outerVar = '외부';

      function inner() {
            let innerVar = '내부';

            console.log(innerVar);  // '내부' (자신)
            console.log(outerVar);  // '외부' (부모)
            console.log(global);    // '전역' (최상위)
      }
      inner();
  }
  outer();
  ```

  * **찾는 순서**
  ```JavaSCript
  1. 자기 자신 스코프
  2. 부모 스코프
  3. 부모의 부모 스코프
  4. ...
  5. 전역 스코프
  6. 없으면 ReferenceError
  ```

  * **예제**
  ```JavaSCript
  let x = 'global';

  function outer() {
      let x = 'outer';

      function inner () {
            let x = 'inner';
            console.log(x);  // 'inner' (가장 가까운 x)
      }

      inner();
      console.log(x);  // 'outer'
  }

  outer();
  console.log(x);  // 'global'
  ```

</details>

<details>
  <summary>(6) 변수 가리기(Variable Shadowing)</summary>

  * **같은 이름의 변수가 여러 스코프에**
  ```JavaScript
  let name = '전역';

  function test() {
      let name = '지역';
      console.log(name);  // '지역' (전역 변수를 가림)
  }

  test();
  console.log(name);  // '전역'
  ```

  * **블록에서도**
  ```JavaScript
  let count = 1;

  if (true) {
      let count = 2;
      console.log(count);  // 2

      if (true) {
          let count = 3;
          console.log(count);  // 3
      }

      console.log(count);  // 2
  }

  console.log(count);  // 1
  ```

</details>

<details>
  <summary>(7) 렉시컬 스코프(Lexical Scope, 정적스코프)</summary>

  * **함수가 정의된 위치에서 스코프 결정**
  ```JavaScript
  let x = 1;

  function func1() {
      let x = 10;
      func2();
  }

  function func2() {
      console.log(x);
  }

  func1();
  ```

  * **동적 스코프와의 차이**
  ```JavaScript
  렉시컬 스코프 (JavaScript)
  - 함수가 정의된 위치에서 스코프 결정
  - 코드를 보면 알 수 있음

  동적 스코프(일부 언어)
  - 함수가 호출된 위치에서 스코프 결정
  - 실행해봐야 알 수 있음
  ```

</details>

<details>
  <summary>(8) 스코프 베스트 프랙티스</summary>

  * **전역 변수 최소화**
  ```JavaScript
  // 나쁜 예: 전역 공간이 지저분
  let data1 = [];
  let data2 = [];
  let data3 = [];

  // 좋은 예: 지역 변수로 캡슐화
  function processData() {
      let data1 = [];
      let data2 = [];
      let data3 = [];
  }
  ```

  * **블록 스코프 활용**
  ```JavaScript
  // 나쁜 예(var)
  for (var i = 0; i < 3; i++) {
      // ...
  }
  console.log(i);  // 3(바깥에서도 접근 가능)

  // 좋은 예(let)
  for (let i = 0;  < 3; i++) {
      // ...
  }
  // console.log(i);  // ReferenceError
  ```

  * **변수는 사용 직전에 선언**
  ```JavaScript
  // 나쁜 예
  function processUser() {
      let name;
      let age;
      let email;
      // ... 100줄 후 사용

  // 좋은 예
  function processUser() {
      // 필요한 시점에 선언
      let name = getUserName();
      console.log(name);

      // 나중에 필요한 건 나중에
      let age = getUserAge();
      console.log(age);
  ```
</details>

# 2. 호이스팅⭐️

<details>
  <summary>(1) 호이스팅이란?</summary>

  * **Hoisting = 끌어올림**
    * 변수의 선언부가 위로 쫙 끌어올려지는 느낌
</details>

<details>
  <summary>(2) var 호이스팅</summary>

  * **선언이 끌어올려짐**
  ```JavaScript
  console.log(name);  // undefined (에러 아님)
  var name = '홍길동';
  console.log(name);  // 홍길동
  ```

  * **실제 동작**
  ```JavaScript
  // JavaScript가 내부적으로 이렇게 해석함
  var name;            // 선언이 맨 위로
  console.log(name);   // undefined
  name = '홍길동';     // 할당은 그대로
  console.log(name);  // 홍길동
  ```

  * **함수 안에서**
  ```JavaScript
  function test() {
      console.log(x);  // undefined
      var x = 10;
      console.log(x);  // 10
  }

  // 실제 동작
  function test() {
      var x;           // 함수 맨 위로
      console.log(x);  // undefined
      x = 10;
      console.log(x);  // 10
  }
  ```
</details>

<details>
  <summary>(3) let과 const 호이스팅</summary>

  * **호이스팅은 되지만 접근 불가**
  ```JavaScript
  console.log(name);  // ReferenceError
  let name = '홍길동';
  ```

  * **TDZ(Temporal Dead Zone)**
  ```JavaScript
  // TDZ 시작
  console.log(name);  // ReferenceError
  console.log(age);   // ReferenceError
  // TDZ
  // TDZ
  let name = '홍길동';  // TDZ 끝
  const age = 25;      // TDZ 끝

  console.log(name);  // 홍길동(정상 작동)
  ```

  * **왜 이렇게 만들었을까?**
  ```JavaScript
  // var의 문제점
  console.log(count);  // undefined (버그 찾기 어려움)
  var count = 10;

  // let/const는 명확하게 에러
  console.log(count);  // ReferenceError (즉시 발견)
  let count = 10;
  ```
</details>

<details>
  <summary>(4) 함수 호이스팅</summary>

  * **함수 선언문 - 전체가 끌어올려짐**
  ```JavaScript
  greet();  // 안녕하세요! (에러 없음)
  function greet() {
      console.log('안녕하세요!');
  }
  ```

  * **실제 동작**
  ```JavaScript
  // 함수 전체가 맨 위로
  function greet() {
      console.log('안녕하세요!');
  }

  greet();  // 안녕하세요!
  ```

  * **함수 표현식 - 변수 규칙을 따름**
  ```JavaScript
  greet();  // TypeError: greet is not a function

  var greet = function() {
      console.log('안녕하세요!');
  };
  ```

  * **실제 동작**
  ```JavaScript
  var greet;        // 선언만 올라감
  greet();          // TypeError(아직 함수 아님)

  greet = function() {
      console.log('안녕하세요!');
  };
  ```
</details>

<details>
  <summary>(5) 호이스팅 예제</summary>

  * **예제 1**
  ```JavaScript
  var x = 1;

  function test() {
      console.log(x);
      var x = 2;
  }

  test();  // undefined (1 아님)

  // 실제 동작
  function test() {
      var x;           // 지역 변수 x 선언이 위로
      console.log(x);  // undefined
      x = 2;
  }
  ```

  * **예제 2**
  ```JavaScript
  function outer() {
      console.log(inner);  // [Function: inner]

      function inner() {
          console.log('내부 함수');
      }
  }

  outer();

  // 함수 선언은 전체가 올라감
  ```

  * **에제 3**
  ```JavaScript
  console.log(add(2, 3));  // 5
  console.log(subtract(5, 2));  // TypeError

  // 함수 선언문 - 호이스팅 O
  function add(a, b) {
      return a + b;
  }

  // 함수 표현식 - 호이스팅 X
  var subtract = function(a, b) {
      return a - b;
  };
  ```
</details>

<details>
  <summary>(6) 호이스팅과 스코프</summary>

  * **함수 스코프 단위로 호이스팅**
  ```JavaScript
  var x = 'global';

  function outer() {
      console.log(x);  // undefined (global 아님)

      if (true) {
          var x = 'local';
      }
      console.log(x);  // local
  }

  outer();
  ```

  * **let은 블록 스코프**
  ```JavaScript
  let x = 'global';

  function outer() {
      console.log(x);  // global (블록 밖의 x)

      if (true) {
          let x = 'local';
      }

      console.log(x);  // global
  }

  outer();
  ```
</details>

<details>
  <summary>(7) 호이스팅 베스트 프랙티스</summary>

  * **변수는 맨 위에 선언**
  ```JavaScript
  function myFunction() {
      let a = 1;
      let b = 2;
      let c = 3;

      // 로직...
  }
  ```

  * **var 사용하지 않기**
  ```JavaScript
  // 나쁜 예
  console.log(name);
  var name = '홍길동';

  // 좋은 예
  let name = '홍길동';
  console.log(name);
  ```

  * **함수는 사용 전에 선언**
  ```JavaScript
  function greet() {
      console.log('안녕하세요!');
  }

  greet();
  ```

  * **함수 표현식 선호**
  ```JavaScript
  // 호이스팅을 명확하게 피하기
  const greet = function() {
      console.log('안녕하세요!');
  };

  greet();
  ```
</details>
